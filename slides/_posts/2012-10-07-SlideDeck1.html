AMD

Asynchronous Module Definition

Asynchronous
Module
Definition

  • AMD is not a library
  • AMD is a specification

With AMD you can...

  • Define async modules
  • Define fine grained dependencies
  • Develop without a build step
  • And a lot more

AMD at its core...


The specification defines a single function "define" that is available as a free variable or a global variable.

define()

define(id?, dependencies?, factory);

For example, myModule.js

define(['yourModule'], function(yourModule){
   // You are in a module called 'myModule'
   // That depends on and is provided 'yourModule'
});

AMD defines how to write modules

Not how to load or use them

View.render()


The heart of any view is its render() method which, by default, does nothing


Yes, nothing.

The sugar


this.el; // The dom element you have
this.$el; // a cached jQuery instance of your el
this.$(); // a jQuery method scoped to your el

View Events


var MyView = Backbone.View.extend({
    events : {
    'click .btn' : 'onButtonClick'
    },
    onButtonClick : function(evt) {
    // automatically called
    }
});

Wiring it all together


Listening and acting on events


var MyView = Backbone.View.extend({
    initialize : function() {
    this.collection.on('all', this.render, this);
    },
    render : function() {
    // Do some rendering magic
    }
});

Voila, a cheap re-render when your collection changes.

Your event management system. Critical for memory management.

Regions & Region Managers

marionette.region

Region managers provide a consistent way to manage your views in your application. They automate the opening, closing, and cleanup of views.


view.region1.show(subView);
view.region1.show(secondSubView);

Serialization of
Models & Collections


Marionette views automatically delegate to the serializeData method before submitting to your templates

Automatic View Cleanup


Marionette's close() method performs the bulk of your cleanup for you.


Using EventBinder allows your views to keep track of listened events to be cleaned up at the end.

Sensible defaults


Marionette lets you change defaults easily but provides some lightly opinionated defaults to get you started.


  • Underscode templates
  • Dom-based templates
  • Data serialization delegating to toJSON()
  • and more

Vastly Reduced Boilerplate

The included views are intended for you to extend and get you off to a killer start.

Marionette.View


A super-enhanced Backbone View.


EventBinder, Backbone Model/Collection binding, UI bindings, View triggers, Data Serialization, Template support.

Marionette.ItemView


A View automating the rendering of Models and Collections into templates.


var OurView = Marionette.ItemView.extend({
    ui : {
        anElement : '.selector'
    }
});

Marionette.CollectionView


A View that automates rendering Collections and child ItemViews.


var OurView = Marionette.CollectionView.extend({
    itemView : SomeView,
    emptyView : anEmptyView
});

Marionette.CompositeView


A useful for complex Collections and Tree-like sets of data.


Recursive by default.

Marionette.Layout


An ItemView extended with region managers.


var OurView = Marionette.Layout.extend({
    regions : {
    main : '.main',
    menu : '.menu'
    }
});

ourView.main.show(subView);


Regions & Layouts allow you to dynamically compose your application's visuals at runtime
